Fragmentation Showcase

This fixture exercises page and column fragmentation: balanced columns, page breaks, and repeating table headers with enough content to span multiple pages.

Balanced Columns

Column balancing keeps the content even across the available columns so that the columns end at roughly the same depth instead of leaving a large gap at the bottom.

Each paragraph here is short to encourage several distinct line boxes and to create clear break opportunities for the column balancer to consider.

A pull-quote with break-inside: avoid-column should never be split between columns. When there is not enough space it should move to the top of the next column set.

Additional filler text keeps flowing after the pull-quote. The balancer should take this into account when deciding where the second column starts, respecting widows and orphans as it fragments the text.

With more text than fits in two balanced columns, the layout should create a second column set while retaining the same column widths and gaps. The inter-column gap is visible thanks to the matching paragraph edges.

Short sentences keep the rhythm predictable. This helps the regression test catch changes in how the fragmentation engine distributes content between columns.

When pagination interacts with multi-column containers the fragmentainer height hint from the page size should keep the columns roughly balanced without leaving trailing whitespace.

Another paragraph to extend the content and ensure that at least two column sets are produced when rendered at the print viewport used by the test harness.

Page Break Before Chapter

This chapter starts on a new page thanks to break-before: page. The text itself runs long enough to occupy a chunk of the page so that subsequent sections are forced onto later pages, providing additional pagination coverage.

Using an explicit break before a chapter heading should insert a blank page if required to satisfy side preferences. The default rules here allow the pagination algorithm to reuse the existing content flow while respecting the break directive.

More filler content keeps the chapter stretching into the page. The goal is simply to increase the amount of material so that repeating table headers in the following section have to cross a fragment boundary.
